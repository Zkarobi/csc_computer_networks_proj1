Hosting a Website with GitHub Pages & Understanding Networking Concepts

Project Overview

This project demonstrates how to set up a website using GitHub Pages while applying key networking concepts like DNS, IP addressing, and protocols.

Steps We Took

  • 1. Set Up a GitHub Repository: Created a public repository for the website.
  • 2. Built the Website: Designed a simple HTML & CSS page.
  • 3. Hosted on GitHub Pages: Enabled GitHub Pages for free website hosting.
  • 4. Configured DNS & IP: GitHub assigns a subdomain and an IP address.
  • 5. Implemented Security: HTTPS is enabled by default for encryption.
  • 6. Verified Networking Aspects: Used `ping` and `traceroute` to test reachability.

Networking Concepts Applied

  • DNS (Domain Name System): GitHub Pages assigns a subdomain (e.g., username.github.io/repository-name). When a user enters the domain, the DNS translates it into an IP address associated with GitHub's hosting servers.
  • IP Addressing: GitHub Pages serves the website from a set of IP addresses. We verified this by using the ping command.
  • HTTP/HTTPS: GitHub Pages enforces HTTPS by default, ensuring secure communication using TLS encryption.
  • Ping & Traceroute:
    • Ping: Running ping username.github.io verifies the site's reachability.
    • Traceroute: Running traceroute username.github.io shows the hops taken to reach GitHubâ€™s servers.
